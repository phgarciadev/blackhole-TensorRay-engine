# Contribuindo para o Black Hole Simulator

Primeiro, obrigado por considerar contribuir. É sério. Mas leia as regras.

## O Jeito de Fazer as Coisas

1.  **Leia a Constituição**: Antes de tudo, leia `.specify/constitution.md` e `.gemini/GEMINI.md`.
2.  **Spec-Driven**: Não abra um PR gigante sem antes abrir uma Issue ou discutir o Spec. Ninguém quer revisar 5000 linhas de código que não resolvem o problema certo.
3.  **Código**: Siga o estilo do Kernel Linux para C. Siga as convenções idiomáticas (mas no_std) para Rust.

## Pull Requests

1.  Preencha o template PULL_REQUEST_TEMPLATE.md completamente.
2.  Se o CI falhar, não peça review.
3.  Seja paciente. Reviews demoram porque nós realmente lemos o código.

## Reportando Bugs

Use o template de Bug Report. Forneça logs. Forneça contexto. Ajude-nos a te ajudar.

## Código de Conduta

Seja excelente uns com os outros. Críticas técnicas são aceitas e encorajadas. Ataques pessoais não. Humor ácido é tolerado e encorajado, desde que não seja ofensivo.
